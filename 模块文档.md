NETE 链上经济模型·前端开发文档

## 一、项目整体定位

NETE 是一个基于区块链、以通缩经济模型与社区自治为核心的“DeFi 收益聚合”赛道卷轴生态。核心叙事是“新范式：重塑时间价值；新引擎：构建去中心化经济引擎”。项目通过链上智能合约实现规则透明与自动执行，融合“质押销毁挖矿”“多级社区激励”与“三重通缩机制”，旨在解决传统卷轴模式“暗箱操作、不可持续”的行业痛点，实现代币价值的长期稳定与增长。

## 三、首页矿机模块展示规则

**展示内容：**

| 展示项 | 展示规则 |
|--------|----------|
| 矿机型号列表 | 展示 100 型、300 型、500 型、1000 型、3000 型、5000 型、10000 型、30000 型、50000 型，按成本金额从小到大排序展示 |
| 每款矿机信息 | 矿机价格（NETE）、周期（天）、周期产量（收益天）、最长周期（天）、提币手续费（%） |
| 矿机数量输入/选择 | 每款矿机旁展示“购买数量”选择器，支持输入数量（受该型号剩余可购买数量限制） |
| 已购买矿机列表 | 展示本人已激活的所有矿机：矿机型号、购买数量、周期进度、已产出收益、剩余周期 |
| 空投矿机 | 单独区域展示空投矿机状态：是否已合成、有效期剩余天数、是否已触发“买一赠一”变永久条件 |

**交互规则：**

- 购买矿机时，扣款优先从**本金钱包**扣除
- 矿机每日产出：本金 + 利润，本金自动进入复投（转入流通钱包或在系统中自动用于复购新矿机）；利润进入利润钱包，可提取或复购
- 矿机每 10 秒产块，每日列表收益（前端展示日收益汇总）

**减产机制提示：**
- 每完成一个周期，统一延长 5 天，最长延长期至 180 天
- 每次链上提币和激活矿机均触发销毁（前端需展示相关提示）

## 四、领导力等级模块（V1–V9）

等级依据**小区业绩（NETE）** 确定，展示不同等级的权益与分红比例：

| 等级 | 小区业绩要求 | 分红比例 | 固定奖励（NETE） |
|------|-------------|---------|-----------------|
| V1 | ≥ 1 万 | 20% | 500 |
| V2 | 3 万 | 17% | 1000 |
| V3 | 10 万 | 14% | 2000 |
| V4 | 35 万 | 12% | 4000 |
| V5 | 110 万 | 10% | 8000 |
| V6 | 360 万 | 10% | 16000 |
| V7 | 1500 万 | 8% | 35000 |
| V8 | 5000 万 | 6% | 65000 |
| V9 | 2 个 V8 | 3% + 全球新增业绩 1% | 100000 |

**前端展示要求：**
- 展示用户当前等级、距离下一等级所需小区业绩差额
- 分红比例为社区每日分红中分配的比例，其中 50% 平分给 V1–V9，50% 按当日新增业绩加权分配
- 固定奖励按等级累计领取说明：V1 领 500 枚，V2 累计领 1000 枚（即新增 500 枚），后续以此类推
- 全球新增业绩 1% 仅 V9 享有

## 五、C2C 模块（C2C / C2C/sell / C2C/market）

### 5.1 C2C 总览

**展示内容：**
- 连接钱包/已连接钱包状态
- 说明文案：“通过 C2C 卖出 NETE 获取 USDT，普通用户通过 C2C 购买的 NETE 只可用于购买矿机进行挖矿”
- 四个核心入口：创建 C2C 卖单 / 查看 C2C 市场 / C2C 委托订单 / C2C 成交订单
- 我的 USDT 余额、我的 NETE 余额

### 5.2 创建卖单

**展示字段：**
- 卖出数量（NETE），支持手动输入或“全部”
- 卖出价格（USDT）
- 我的 USDT 余额、我的 NETE 余额（实时刷新）

**规则提示：**
- 仅支持挂卖单
- 挂单生成后，10 分钟内为定向交易（仅指定地址可见/可交易），超时后转为全网可见
- 手续费率：10%（由卖方支付 USDT）
- 24 小时未成交时，官方可按托底条件回购

**做市商说明：**
- V4 及以上等级可申请做市商资格
- 做市商独立账户可免手续费，赚取 5% 的市场差价

### 5.3 C2C 市场

**展示内容：**
- 当前所有活跃卖单（卖方、数量、单价、剩余时间）
- 按单价排序（由低到高）
- 每个卖单旁显示“购买”按钮

**交互规则：**
- 前端用户点击购买跳转至卖单详情或支付确认页
- 交易的 NETE 仅可用于购买矿机挖矿，不可二次 C2C 转卖

### 5.4 委托订单与成交订单

| 类型 | 展示内容 |
|------|----------|
| C2C 委托订单 | 用户已挂出但未成交的卖单列表，支持“撤单”操作 |
| C2C 成交订单 | 用户已成交的卖单/买单历史列表，展示时间、数量、单价、成交额 |

## 十、提币手续费分配说明（全局展示）

提币手续费率按矿机类型收取（20%–30%），分配方式如下：

| 分配项 | 占比 | 用途 |
|--------|------|------|
| 销毁 | 20% | 直接打入黑洞地址，实现通缩 |
| 项目方 | 30% | 运营、开发、市场 |
| 社区分红 | 50% | 每日自动分配，其中 50% 平分给 V1–V9，50% 按当日新增业绩加权分配 |

前端需在提币确认页面展示该手续费与分配明细。

## 十一、销毁与通缩机制展示（全局说明）

- **初始总量**：30 亿枚 NETE（恒定永不增发）
- **终极通缩目标**：流通量压缩至 2100 万枚
- **通缩方式**：每一次链上提币和激活矿机均触发销毁
- **展示信息**：全局角标显示“已销毁总量”“当前流通总量”“剩余销毁至目标所需数量”

## 十三、前端开发关键技术要求（概要）

| 维度 | 要求 |
|------|------|
| 钱包连接 | 支持 Web3 钱包（如 MetaMask），连接后识别链上地址 |
| 数据来源 | 所有核心数据均从链上合约读取，无中央数据源 |
| 实时刷新 | 钱包余额、矿机状态、价格需支持轮询或 WebSocket 实时更新 |
| 销毁事件 | 前端展示销毁事件记录，支持按时间线筛选 |
| 分红记录 | 前端展示社区分红历史与分配明细 |
| 权限控制 | V1–V9 等级权益需根据链上业绩自动判定与展示 |
| 多语言 | 支持中英文双语切换（页面文案国际化） |
| 响应式 | 支持 PC 端与移动端 H5 适配 |
